# 中文公文格式规范技能

## 核心原则

> ⚠️ **最重要规则：只修改格式，绝不修改、删除或增加任何文件内容。**
> 所有原文文字保持原样，仅对字体、字号、间距、对齐等格式进行调整。

-----

## 通用格式规范

### 页面设置

- 纸张：A4（210mm × 297mm）
- 页边距（默认）：上 3.7cm，下 3.5cm，左 2.8cm，右 2.6cm

### 单页排版原则

> ✅ **尽量使文件内容在同一页内完整呈现，避免内容溢出到第二页。**

当内容接近或超出一页时，按以下顺序依次尝试压缩，直到内容适配一页为止：

|优先级|调整项     |允许范围                 |
|---|--------|---------------------|
|1  |缩小页边距   |上下最小 2.0cm，左右最小 1.8cm|
|2  |缩小行距    |最小固定值 24pt（不低于此值）    |
|3  |减少段落间距  |段前段后可设为 0            |
|4  |缩小落款前空行数|最少保留1行（原则是3行，可压缩至1行） |

**注意：**

- 调整页边距时，左右边距保持对称
- 字体和字号不得因单页排版需要而缩小，字体规范优先于单页原则
- 如内容确实无法在一页内呈现，以内容完整为准，不强制压缩

**XML 页边距设置（DXA单位，1440 DXA = 1 inch ≈ 2.54cm）：**

```xml
<!-- 默认页边距 -->
<w:pgMar w:top="2126" w:right="1474" w:bottom="1985" w:left="1587"/>

<!-- 压缩版页边距（上下2.0cm，左右1.8cm） -->
<w:pgMar w:top="1134" w:right="1021" w:bottom="1134" w:left="1021"/>
```

常用换算：1cm ≈ 567 DXA

### 字体总览

|位置           |字体      |字号      |
|-------------|--------|--------|
|标题           |方正小标宋GBK|二号（22pt）|
|一级标题（一、二、三…） |黑体      |3号（16pt）|
|二级标题（（一）（二）…）|楷体GB2312|3号（16pt）|
|正文           |仿宋GB2312|3号（16pt）|
|落款/日期        |仿宋GB2312|3号（16pt）|

### 间距规范

- 标题与正文之间：空1行
- 正文与落款之间：空3行
- 正文行距：固定值 28pt（或1.5倍行距）
- 段落间距：段前0行，段后0行

### 对齐规范

- 标题：居中对齐
- 正文：两端对齐（首行缩进2字符）
- 称谓行（如”×××部门：”）：顶格，不缩进
- 落款/日期：右对齐（或根据模板居中右侧区域）

## 各类文件格式规范

### 1. 函文件（邀请函/商函）

```
结构顺序：
1. 标题（方正小标宋GBK 二号，居中）
2. 空1行
3. 称谓行（仿宋GB2312 3号，顶格，如"×××："）
4. 正文段落（仿宋GB2312 3号，首行缩进2字符）
5. 结束语（如"专此函达，敬请回复。"）
6. 空3行
7. 落款（仿宋GB2312 3号，右侧）
8. 日期（仿宋GB2312 3号，右侧，格式：XXXX年X月X日）
```

**关键格式点：**

- 标题不加句号
- 称谓后加冒号”：”，单独一行，顶格
- 结束语单独成段，首行缩进2字符
- 落款和日期右对齐

-----

### 2. 会议通知

```
结构顺序：
1. 标题（方正小标宋GBK 二号，居中）
2. 空1行
3. 一、会议时间（黑体 3号）
   正文（仿宋GB2312 3号）
4. 二、会议地点
5. 三、会议主持人
6. 四、参会人员
   （一）（楷体GB2312 3号）
   （二）（三）…
5. 五、会议议程
6. 六、注意事项
   1.（仿宋GB2312 3号）
7. 空3行
8. 单位落款（仿宋GB2312 3号）
9. 日期
```

**关键格式点：**

- 一级条目”一、二、三…“用黑体3号
- 二级条目”（一）（二）…“用楷体GB2312 3号
- 数字条目”1. 2. 3.“用仿宋GB2312 3号

### 3. 会议纪要

结构顺序：
1. 标题（方正小标宋GBK 二号，居中）
2. 空1行
3. 一、会议时间（黑体 3号）
   正文（仿宋GB2312 3号）
4. 二、会议地点
5. 三、参加人员
6. 四、列席人员
7. 五、主持人
8. 六、记录人

10. 八、会议内容
    （一）（楷体GB2312 3号）
    1. 正文（仿宋GB2312 3号）
11. 空3行
12. 落款（仿宋GB2312 3号）
13. 日期
```

### 4. 活动方案申请

```
结构顺序：
1. 标题（方正小标宋GBK 二号，居中）
2. 空1行
3. 一、活动基本信息（黑体 3号）
   （一）活动主题（楷体GB2312 3号）
   （二）活动时间
   （三）活动地点
   （四）参与对象
4. 二、活动目的
   正文（仿宋GB2312 3号）
5. 三、活动内容 … （以此类推各节）

### 5. 申请类文件

```
结构顺序：
1. 标题（方正小标宋GBK 二号，居中）
2. 空1行
3. 称谓行（×××部门：顶格，仿宋GB2312 3号）
4. 正文段落
5. 一、活动方案内容（黑体 3号）
   （一）活动名称（楷体 3号）
   （二）活动时间
   （三）活动内容
6. 二、费用明细（黑体 3号）
   1. 条目（仿宋GB2312 3号）

8. 空3行
9. 申请部门落款

```

## 操作流程（格式修改步骤）

1. **读取技能文档**：先读取 `/mnt/skills/public/docx/SKILL.md` 了解 docx 操作方法
1. **识别文件类型**：根据标题和内容判断是哪类公文（函件/通知/纪要/方案/申请）
1. **解包文件**：`python scripts/office/unpack.py input.docx unpacked/`
1. **逐项修改格式**（严格按对应类型规范）：
- 标题字体、字号、对齐
- 各级条目字体
- 正文字体
- 段间距、行间距
- 落款/日期格式
1. **重新打包**：`python scripts/office/pack.py unpacked/ output.docx --original input.docx`
1. **验证**：`python scripts/office/validate.py output.docx`

> ⚠️ 编辑 XML 时，**只修改 `<w:rPr>`（字体/字号）和 `<w:pPr>`（段落格式）节点**，
> 绝对不能修改 `<w:t>` 标签内的文字内容。

## XML 格式参考

### 字体对应关系（XML中的字体名称）

|字体名称    |XML 中的值                                   |
|--------|------------------------------------------|
|方正小标宋GBK|`FangZheng XiaoBiaoSong GBK` 或 `方正小标宋_GBK`|
|仿宋GB2312|`FangSong_GB2312` 或 `仿宋_GB2312`           |
|黑体      |`SimHei` 或 `黑体`                           |
|楷体GB2312|`KaiTi_GB2312` 或 `楷体_GB2312`              |

### 字号对应关系（半点单位，1pt = 2单位）

|中文字号|pt  |XML `w:sz` 值|
|----|----|------------|
|二号  |22pt|44          |
|3号  |16pt|32          |

### 段落格式示例

```xml
<!-- 标题段落：方正小标宋GBK 二号，居中 -->
<w:pPr>
  <w:jc w:val="center"/>
  <w:rPr>
    <w:rFonts w:hint="eastAsia" w:eastAsia="方正小标宋_GBK" w:ascii="方正小标宋_GBK"/>
    <w:sz w:val="44"/>
    <w:szCs w:val="44"/>
  </w:rPr>
</w:pPr>

<!-- 一级标题：黑体 3号 -->
<w:pPr>
  <w:rPr>
    <w:rFonts w:hint="eastAsia" w:eastAsia="黑体" w:ascii="黑体"/>
    <w:b/>
    <w:sz w:val="32"/>
    <w:szCs w:val="32"/>
  </w:rPr>
</w:pPr>

<!-- 正文段落：仿宋GB2312 3号，首行缩进2字符 -->
<w:pPr>
  <w:ind w:firstLine="640"/>
  <w:rPr>
    <w:rFonts w:hint="eastAsia" w:eastAsia="仿宋_GB2312" w:ascii="仿宋_GB2312"/>
    <w:sz w:val="32"/>
    <w:szCs w:val="32"/>
  </w:rPr>
</w:pPr>
```

### 段间距（标题后空1行 / 落款前空3行）

```xml
<!-- 标题后空1行 -->
<w:spacing w:after="320"/>

<!-- 正文末尾，落款前空3行 -->
<!-- 插入3个空行段落，字体仿宋 3号 -->
<w:p><w:pPr><w:rPr><w:sz w:val="32"/></w:rPr></w:pPr></w:p>
<w:p><w:pPr><w:rPr><w:sz w:val="32"/></w:rPr></w:pPr></w:p>
<w:p><w:pPr><w:rPr><w:sz w:val="32"/></w:rPr></w:pPr></w:p>
```

## 常见错误检查清单

格式修改完成后，检查以下项目：

- [ ] 标题是否为方正小标宋GBK 二号，居中
- [ ] 标题与正文之间是否空1行
- [ ] 一级条目是否为黑体3号
- [ ] 二级条目是否为楷体GB2312 3号
- [ ] 正文是否为仿宋GB2312 3号，首行缩进2字符
- [ ] 落款/日期前是否空3行
- [ ] 落款/日期字体是否为仿宋GB2312 3号
- [ ] 文字内容是否完全未被修改
- [ ] 内容是否在同一页内完整呈现；如有溢出，是否已按优先级调整页边距/行距
